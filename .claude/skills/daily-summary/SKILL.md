# Daily Summary

오늘 자정 이후의 git commit 기록을 수집하고, 변경 내용을 분석하여 주제별로 묶은 구조화된 정리본을 생성한다.

## 실행 절차

### 1. 커밋 수집

```bash
git log --since="midnight" --format="%H %s" --reverse
```

커밋이 없으면 "오늘 커밋 기록이 없습니다."라고 알리고 종료한다.

### 2. 변경 분석

각 커밋에 대해 변경 내용을 파악한다:

```bash
git show --stat <hash>   # 변경된 파일 목록
git show <hash>          # 전체 diff
```

커밋 메시지와 실제 diff를 모두 읽어서, 단순 파일 목록이 아닌 **무엇을 왜 변경했는지** 핵심을 파악한다.

### 3. 주제 분류

커밋들을 **시간순이 아닌 의미 단위(주제/기능)**로 묶는다. 예를 들어:
- CSS 스타일링 관련 커밋 3개 → 하나의 주제
- JS 기능 추가 커밋 2개 → 하나의 주제
- 배포/설정 관련 커밋 → 별도 주제

### 4. 정리본 출력

아래 양식에 맞춰 stdout에 출력한다.

### 5. 마크다운 파일 저장

정리본 출력 후, 동일한 내용을 마크다운 파일로 저장한다.

- 파일 경로: `summaries/YYYY-MM-DD.md` (오늘 날짜 기준)
- `summaries/` 디렉토리가 없으면 생성한다
- 같은 날짜 파일이 이미 존재하면 덮어쓴다
- 저장 완료 후 파일 경로를 안내한다

## 출력 양식

```
# 📌 N. 주제명

## N-1. 대제목
> 💡 핵심 요약 한 줄

### 1) 소제목
- 내용
- 내용
- 내용

### 2) 소제목
- 내용
```

### 번호 체계
- `N` — 대주제 번호 (1, 2, 3...)
- `N-1` — 대주제 내 하위 번호 (1-1, 1-2, 2-1, 2-2...)
- `1), 2)` — 소제목 순번

### 작성 규칙
- 소제목 하나당 내용 **3줄 내외**로 압축. 단, 날것 메모 입력 시 항목 수는 원본을 따름
- 의미상 연관된 것끼리 묶어 `### 소제목` 구성
- 이미지/사진 관련 변경도 동일 양식 적용
- 이모티콘은 `💡`만 사용 (다른 이모티콘 절대 금지. 📌는 최상위 제목에만 허용)
- **변경 내용 누락 없이 전부 포함**
- 전체 내용은 **한국어**로 작성
- 코드 변경의 의미를 파악하여 기술적 내용을 이해하기 쉽게 풀어쓴다
