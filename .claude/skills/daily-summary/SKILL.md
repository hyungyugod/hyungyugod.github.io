# Daily Summary

오늘 하루의 git commit 기록을 수집하고, 변경 내용을 분석하여 주제별로 묶은 구조화된 정리본을 생성한다.

## 날짜 기준

**새벽 4시를 하루의 경계**로 삼는다.
- 현재 시각이 00:00~03:59이면 **전날**로 취급한다 (예: 4월 3일 02:00 → 4월 2일로 간주)
- 현재 시각이 04:00~23:59이면 **당일**로 취급한다

커밋 수집 범위도 이 기준을 따른다:
- 기준 날짜 당일 04:00부터 다음 날 03:59까지의 커밋을 수집한다

## 실행 절차

### 1. 커밋 수집

먼저 현재 시각을 확인하여 기준 날짜를 결정한다:

```bash
# 현재 시각 확인
date +%H

# 04시 이전이면 전날을 기준 날짜로 사용
# 04시 이후이면 당일을 기준 날짜로 사용
```

기준 날짜가 정해지면 해당 범위의 커밋을 수집한다:

```bash
# 기준 날짜 04:00 ~ 다음 날 03:59 범위
git log --after="YYYY-MM-DD 04:00" --before="YYYY-MM-DD+1 04:00" --format="%H %s" --reverse
```

커밋이 없으면 "해당 날짜의 커밋 기록이 없습니다."라고 알리고 종료한다.

### 2. 변경 분석

각 커밋에 대해 변경 내용을 파악한다:

```bash
git show --stat <hash>   # 변경된 파일 목록
git show <hash>          # 전체 diff
```

커밋 메시지와 실제 diff를 모두 읽어서, 단순 파일 목록이 아닌 **무엇을 왜 변경했는지** 핵심을 파악한다.

### 3. 주제 분류

커밋들을 **시간순이 아닌 의미 단위(주제/기능)**로 묶는다. 예를 들어:
- CSS 스타일링 관련 커밋 3개 → 하나의 주제
- JS 기능 추가 커밋 2개 → 하나의 주제
- 배포/설정 관련 커밋 → 별도 주제

### 4. 정리본 출력

아래 양식에 맞춰 stdout에 출력한다.

### 5. 마크다운 파일 저장

정리본 출력 후, 동일한 내용을 마크다운 파일로 저장한다.

- 파일명 양식: `summaries/YYYY-MM-DD-(제목).md` (기준 날짜 사용)
- 제목은 정리본의 핵심 내용을 **한국어 2~4단어**로 요약하여 괄호 안에 넣는다 (예: `2026-04-02-(UI-애니메이션-강화).md`)
- 제목에 공백 대신 `-`(하이픈)을 사용한다
- `summaries/` 디렉토리가 없으면 생성한다
- 같은 날짜·제목 파일이 이미 존재하면 덮어쓴다
- 저장 완료 후 파일 경로를 안내한다

## 출력 양식

```
# 📌 N. 주제명

## N-1. 대제목
> 💡 핵심 요약 한 줄

### 1) 소제목
- 내용
- 내용
- 내용

### 2) 소제목
- 내용
```

### 번호 체계
- `N` — 대주제 번호 (1, 2, 3...)
- `N-1` — 대주제 내 하위 번호 (1-1, 1-2, 2-1, 2-2...)
- `1), 2)` — 소제목 순번

### 작성 규칙
- 소제목 하나당 내용 **3줄 내외**로 압축. 단, 날것 메모 입력 시 항목 수는 원본을 따름
- 의미상 연관된 것끼리 묶어 `### 소제목` 구성
- 이미지/사진 관련 변경도 동일 양식 적용
- 이모티콘은 `💡`만 사용 (다른 이모티콘 절대 금지. 📌는 최상위 제목에만 허용)
- **변경 내용 누락 없이 전부 포함**
- 전체 내용은 **한국어**로 작성
- 코드 변경의 의미를 파악하여 기술적 내용을 이해하기 쉽게 풀어쓴다
