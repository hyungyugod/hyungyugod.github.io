배포 프로세스를 실행합니다.

1. **변경 확인**: `git status`와 `git diff`로 변경 내용 요약을 보여준다
2. **정적 검증**: index.html에서 참조하는 파일(CSS, JS, 이미지)이 실제로 존재하는지 확인한다
   - 누락된 파일이 있으면 보고하고 즉시 중단. commit/push 하지 않음
3. **검증 통과 시**: 아래 순서로 실행
   - `git add -A` (변경된 파일 전체 스테이징)
   - `git diff --staged`로 변경 내용을 분석하여 적절한 한국어 커밋 메시지 자동 작성
   - `git commit`
   - `git push origin <현재브랜치>`
4. push 완료 후 결과 요약 출력 (변경 파일 수, 커밋 메시지, 푸시된 브랜치)
