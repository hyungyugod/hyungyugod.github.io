## AI 슬롭 패턴 — 자동 감점 목록

다음 패턴 발견 시 해당 항목에서 -1점 적용.
이 패턴들은 창의적 판단 없이 LLM이 기계적으로 선택하는 "기본 옵션"이다.

| 패턴 | 감점 항목 | 이유 |
|---|---|---|
| 보라-청록 그라디언트(`#8b5cf6`→`#06b6d4`)를 아무 이유 없이 카드/버튼에 적용 | D2 독창성 | 가장 흔한 "AI 디자인" 패턴 |
| `box-shadow: 0 20px 60px rgba(0,0,0,0.3)` 이상 — 모든 카드에 일괄 과대 그림자 | D1 품질 | 맥락 없는 과잉 효과 |
| 사이트 `--radius` 변수 무시, 임의 `border-radius: 20px` 이상 사용 | D3 패턴 | 일관성 파괴 |
| 기존 카드에 scale 효과가 있는데 `transform: scale(1.05)` 이상 중복 추가 | D1 품질 | 과잉 중복 효과 |
| SPEC에 없는 독립 기능을 사전 합의 없이 추가 | 기능완성도 | 범위 위반 |
| `transitionend` 대신 `setTimeout(fn, 300)`으로 애니메이션 완료 처리 | 패턴 일관성 | 취약한 타이밍 |

> **적용 규칙**: SPEC.md의 변경 유형이 **디자인(시각·스타일·애니메이션)** 위주이면 아래 "디자인 변경 평가 기준"을 사용하고, **로직·기능** 위주이면 "기능 변경 평가 기준"을 사용한다.  
> 두 성격이 혼합된 경우 기능 평가 기준을 적용한다.

## 디자인 변경 평가 기준

시각 디자인, 스타일, 애니메이션, 레이아웃 조정 등 **로직 변경이 없는** 작업에 적용.

### D1. 디자인 품질 (비중: 30%)
변경된 UI가 완성도 높고, 기존 사이트와 시각적으로 조화를 이루는가?

합격: 글래스모피즘 톤 유지, 여백·정렬 일관, 트랜지션 자연스러움, hover/focus 상태 명확, 다크/라이트 테마 모두 대응  
불합격: 색상·크기 튀어 보임, 기존 컴포넌트와 스타일 불일치, 트랜지션 누락, 테마 한쪽만 동작

### D2. 독창성 (비중: 30%)
단순 복사·붙여넣기가 아닌 창의적인 시각 표현인가?

합격: 기존에 없던 인터랙션 또는 시각 효과, 사이트 정체성(glassmorphism + 보라빛 팔레트)을 살린 독자적 아이디어  
불합격: 흔한 부트스트랩 스타일 그대로, 차별화 없는 변경, 기존 컴포넌트 복사 수준

### D3. 패턴 일관성 (비중: 20%)
CSS 변수·BEM·네이티브 중첩 등 코드베이스 규칙을 지키는가?

합격: CSS 커스텀 프로퍼티 사용, BEM 클래스, `&` 네이티브 중첩, 하드코딩 색상 없음  
불합격: 하드코딩 색상, 인라인 스타일, SCSS 문법 혼입, BEM 미준수

### D4. 반응형 & 접근성 (비중: 15%)
모바일(520px 브레이크포인트)에서 깨지지 않고, 접근성 요건을 최소 충족하는가?

합격: 520px 대응, `prefers-reduced-motion` 처리, 포커스 가시성 유지  
불합격: 모바일 레이아웃 깨짐, 애니메이션 모션 설정 무시, 포커스 스타일 제거

### D5. 기능 보전 (비중: 5%)
디자인 변경으로 인해 기존 기능이 손상되지 않았는가?

합격: 기존 JS 동작 정상, 클릭·호버 이벤트 유지, DOM 구조 의도치 않게 변경 없음  
불합격: 기존 기능 동작 안 함, JS 이벤트 바인딩 끊김

---

### 디자인 판정

가중 점수 = (D1×0.30) + (D2×0.30) + (D3×0.20) + (D4×0.15) + (D5×0.05)

- 7.0 이상 → 합격
- 5.0~6.9 → 조건부 합격 (피드백 반영 후 재검수)
- 5.0 미만 → 불합격

### 점수 기준선 (채점 전 비교하라)

**D1 디자인 품질**
- 8점: glassmorphism 톤 유지, hover/focus 명확, 다크/라이트 모두 대응. 한 효과가 약간 강함.
- 5점: 색상 계열 맞으나 간격/폰트가 기존과 불일치. 트랜지션 있으나 어색함.

**D2 독창성**
- 8점: 사이트 glassmorphism + 보라빛 정체성을 활용한 고유 인터랙션 1개 이상.
- 5점: 어디서나 볼 수 있는 패턴 조합. 사이트 정체성과 연결된 아이디어 없음.

**D3 패턴 일관성**
- 8점: BEM/CSS변수/네이티브중첩 준수. P2 수준 사소한 불일치 1-2건.
- 5점: 하드코딩 색상 2건 이상, 또는 BEM 위반 2건 이상.

## 기능 변경 평가 기준

로직 추가·수정, 새 기능 구현 등 **코드 동작이 바뀌는** 작업에 적용.

### 1. 패턴 일관성 (비중: 40%)
기존 코드베이스의 규칙과 패턴을 따르는가?

합격: BEM 네이밍 준수, CSS 변수 사용, 네이티브 중첩 `&` 문법, JS init 패턴, 가드 클래스, 기존 컴포넌트와 동일한 구조
불합격: 하드코딩 색상, SCSS 문법 혼입, 인라인 스타일, BEM 미준수, 기존 패턴과 다른 구조

### 2. 보안 & 접근성 (비중: 25%)
보안 취약점이 없고, 접근성을 충족하는가?

합격: `esc()`/`safeUrl()` 적용, `rel="noopener"`, aria 속성, 포커스 트랩, 키보드 접근, `prefers-reduced-motion` 대응
불합격: XSS 가능, esc() 누락, 인라인 이벤트 핸들러, 접근성 속성 누락, 모달 포커스 트랩 미구현

### 3. 반응형 & UI 품질 (비중: 20%)
모바일과 데스크톱에서 모두 잘 작동하는가?

합격: 520px 브레이크포인트 대응, 간격/패딩 적절, 호버 효과 일관, 트랜지션 자연스러움, 글래스모피즘 일관
불합격: 모바일 레이아웃 깨짐, 호버 효과 누락, 트랜지션 불일치, -webkit- 접두사 누락

### 4. 기능 완성도 (비중: 15%)
SPEC.md에 명시된 기능이 모두 동작하는가?

합격: 모든 기능 구현, 에러 처리 포함, DOMContentLoaded 등록, 정상 동작
불합격: 기능 누락, 미완성 코드, 에러 처리 없음, 초기화 미등록

## 판정

가중 점수 = (패턴×0.4) + (보안접근성×0.25) + (반응형UI×0.2) + (기능×0.15)

- 7.0 이상 → 합격
- 5.0~6.9 → 조건부 합격 (피드백 반영 후 재검수)
- 5.0 미만 → 불합격
- 보안(P0 이슈) 존재 시 → 무조건 불합격

### 점수 기준선 (채점 전 비교하라)

**패턴 일관성**
- 8점: BEM/CSS변수/네이티브중첩/JS init 패턴 준수. P2 불일치 1-2건 있음.
  (실제 예: 하드코딩 P1 2건 + P2 2건 → 7점)
- 5점: 하드코딩 색상 3건 이상, 또는 함수 선언 방식 일관성 없음.

**보안 & 접근성**
- 8점: esc()/safeUrl() 적용, aria 속성, prefers-reduced-motion 대응. 사소한 누락 1건.
- 5점: esc() 누락 1건 또는 접근성 속성 다수 누락.
